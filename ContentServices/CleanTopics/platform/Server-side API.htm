This section describes server-side APIs that can be used in formulas , formula fields , and triggers . See the heading topic for each set of APIs to see where they can be used. Access control permissions apply to server-side APIs that view, edit, create, or delete records. Each API that requires access control states which type of permission is required. Permissions are checked for: The current user (Portal User or Portal Guest role for portals) The administrative role Server API The Server API role is required for bulk jobs and delayed triggers, because there is no user context for these operations. For more information about security and access permissions, see Security and access control .
