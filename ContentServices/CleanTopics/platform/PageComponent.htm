Purpose PageComponent is a client-side JavaScript object that represents a component on an application page. Charts, grids, and tabular reports are all represented as PageComponent objects in the new UI. You can use a PageComponent object's interface to refresh an individual PageComponent or to access its Kendo configuration object. Use the client-side function rbf_getPageComponent() to access a specific PageComponent. Note: The methods in this section apply to the new UI, not the classic UI. Most Platform tenants are using the new UI. However, tenants on older Private Cloud installations might still be using the classic UI. These methods are not supported in portals. Interface You can refresh an individual PageComponent without refreshing its entire page. Use the following methods to refresh a PageComponent: refresh() — Refreshes an individual page component on an application page. For a report, refreshes both the chart and the grid rendered as part of the report. refreshChart() — For a report, refreshes only the chart rendered as part of the report refreshGrid() — For a report, refreshes only the grid rendered as part of the report You can access the Kendo configuration object for a grid or a report using the following method: getKendoConfig() — For a grid, returns the Kendo configuration object for the grid.
