With the appropriate license, you can distribute Platform components across multiple hosts. Your license specifies the number of hosts that can run Platform software and must include the domain name provided by your web server as the host name. To set up an example architecture, follow these steps: Installing NGINX and Configuring NGINX Use the multi-server license provided by Infinite Blue for the Platform Master Server installation. Install the components on their respective hosts. Specify the component details for the Master Server in the node-config.json to include entries for all system components. Note: Ensure that the server URL of each specific node is mentioned in the node-config.json of that machine. Start the database(s). Start the Web server that is hosting the Platform Master Server instance. Start the Web server on the hosts for production components. Navigate to the System Console application System tab to verify that all of your components can be monitored from the Master Server in the Components and Servers tab. Make sure that production servers have the correct parameters.
